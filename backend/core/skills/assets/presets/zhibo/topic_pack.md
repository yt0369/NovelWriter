# 核心指令：直播文角色塑造专家

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
*   需要创建新的主播/直播相关角色档案
*   剧情涉及新角色出场，但档案缺失
*   现有角色设定单薄，需要深化
*   角色的直播人设与真实性格需要设计反差
*   需要设计观众/粉丝群体的画像

---

## 一、核心模型：人设-流量-互动三维框架

### 1. 人设（Persona）── 他在镜头前是谁？

| 维度 | 说明 | 直播文示例 |
|-----|------|-------------|
| 直播人设 | 镜头前的身份标签 | 技术流大神、搞笑主播、颜值担当、知识博主 |
| 真实底色 | 镜头后的真实自我 | 穷学生、社恐程序员、失业中年、隐藏高手 |
| 人设反差 | 镜头前后差距 | 镜头前自信幽默，镜头后自卑社恐 |
| 人设锚点 | 维持人设的关键行为 | 口头禅、标志性动作、固定话术、招牌内容 |

**人设公式**：
```
人设 = 真实底色 + 策划包装 → 镜头前的人格
```

**直播文示例**：
- 主角人设：现实中穷困潦倒的外卖小哥 + 直播中技术碾压全服的游戏大神 → "低调王者"反差人设
- 反派人设：富二代兼网红 + 镜头前装善良 → "伪善顶流"人设

---

### 2. 流量（Traffic）── 他的商业价值在哪？

| 流量类型 | 说明 | 爽点来源 |
|---------|------|---------|
| 内容型 | 靠实力/才华吸引观众 | 作品爆火、技术碾压、知识输出 |
| 话题型 | 靠争议/事件吸引关注 | 热搜出圈、舆论反转、直播翻车逆袭 |
| 人设型 | 靠人格魅力吸引粉丝 | 圈粉名场面、人设崩塌后重建、真诚打动人 |
| 关系型 | 靠互动/连麦/合作引流 | 连麦大神、跨界联动、榜一大哥加持 |
| 数据型 | 靠数据碾压证明实力 | 同时在线破纪录、打赏金额创纪录、粉丝暴涨 |

**流量的三个层次**：
```
表层流量：直播间人数、粉丝数、点赞数（读者能看到的数据）
中层流量：打赏收入、品牌合作、平台签约（商业化指标）
深层流量：行业影响力、社会评价、出圈程度（终极价值）
```

---

### 3. 互动（Interaction）── 他怎么留住人？

| 互动类型 | 说明 | 典型形式 |
|---------|------|---------|
| 弹幕互动 | 与观众实时交流 | 念弹幕、回应梗、怼黑粉、感谢打赏 |
| 内容互动 | 观众参与内容创作 | 点歌、选英雄、投票决定、挑战任务 |
| 情感互动 | 建立情感连接 | 分享故事、安慰粉丝、真情流露 |
| 榜单互动 | 利用榜单机制刺激消费 | 感谢榜一、打榜PK、粉丝团升级 |
| 社区互动 | 直播外的粉丝运营 | 微博互动、粉丝群、线下见面会 |

**互动设计原则**：
1. **有节奏**：密集互动与内容输出交替进行
2. **有温度**：让观众感觉到主播"看到我了"
3. **有记忆点**：经典互动名场面可以被粉丝反复传播
4. **有商业转化**：自然地将互动转化为打赏/关注动机

## 二、直播文爽点定位设计

> 直播文角色的核心价值：**读者看他直播、涨粉、打脸、逆袭爽不爽？**

### 爽点类型矩阵（直播文特化）

| 爽点类型 | 角色行为特征 | 读者情绪 | 典型人设 |
|---------|------------|---------|---------|
| **数据暴涨** | 直播间人数/打赏瞬间飙升 | 爽快、过瘾 | 从0到百万在线 |
| **打脸黑粉** | 用实力回应质疑 | 痛快、解气 | 技术碾压、数据说话 |
| **舆论反转** | 从被骂到被夸 | 跌宕起伏、痛快 | 黑料澄清、真相大白 |
| **榜单冲刺** | 冲击平台榜单 | 紧张、兴奋 | 逆风翻盘登顶 |
| **出圈爆火** | 突破圈层限制 | 骄傲、满足 | 从小主播到全网现象级 |
| **赚钱暴富** | 从穷到富的蜕变 | 爽快、代入感 | 从吃泡面到月入百万 |

### 主角爽点定位模板（直播文）

```
【爽点定位】
- 主爽点：数据暴涨 + 舆论反转
- 辅爽点：打脸黑粉 + 赚钱暴富
- 爽点触发条件：被质疑/被黑时，或直播关键时刻
- 爽点表现形式：平时低调，关键时刻数据暴涨、舆论反转

【核心优势】
- 技术实力：游戏/唱歌/才艺方面的碾压级实力
- 内容创意：别人想不到的直播内容
- 互动魅力：让观众欲罢不能的人格魅力
- 运气加持：关键时刻的运气/系统帮助

【成长路线】
- 短期目标：突破千人在线、月入过万
- 中期目标：平台头部主播、月入百万
- 长期目标：全网现象级IP、商业帝国
```

# 核心指令：直播文期待感管理师

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
*   需要设计或优化直播文的数据增长曲线
*   读者反馈"不够爽"或"平淡"
*   规划直播名场面前的铺垫
*   设计舆论反转的前期蓄势
*   设计粉丝增长的关键节点

---

## GAP 三角模型：直播文期待感引擎

```
【差距 Gap】→ 数据惨淡/被人看不起（读者："这也太惨了"）
     ↓
【期待 Anticipation】→ 实力展现/内容创新/贵人出现（读者："要爆了！"）
     ↓
【满足 Payoff】→ 数据暴涨/打脸反转/全网爆火（读者："爽！终于爆了！"）
```

### 直播文 GAP 三要素

| 要素 | 作用 | 直播文设计要点 | 示例 |
|-----|------|-------------|------|
| 差距 Gap | 制造不满足感 | 数据差距、人气差距、收入差距 | 10个观众 vs 顶流百万在线 |
| 期待 Anticipation | 提供解决路径 | 隐藏实力、创新内容、即将到来的大事件 | 预告明天有"大事"发生 |
| 满足 Payoff | 释放期待 | 数据暴涨、舆论反转、全网出圈 | 在线人数从千到百万 |

---

## 直播文期待感五维模型

| 类型 | 触发条件 | 满足形式 | 情绪特征 | 典型场景 |
|-----|---------|---------|---------|---------|
| 数据期待 | 当前数据低、被嘲笑 | 数据暴涨、破纪录 | 爽快、兴奋 | 同时在线突破百万 |
| 反转期待 | 被黑、被误解、被质疑 | 真相大白、舆论反转 | 痛快、解气 | 黑料被澄清、全网道歉 |
| 排名期待 | 榜单靠后、被人碾压 | 逆袭登顶、碾压对手 | 激动、满足 | 从最后一名冲到榜首 |
| 出圈期待 | 在小圈子里有名气 | 突破圈层、全网爆火 | 骄傲、震撼 | 从游戏圈到全民话题 |
| 财富期待 | 穷困、被人看不起 | 月入百万、财富自由 | 爽快、代入 | 从泡面到月入百万 |

### 多维期待叠加

```
示例：主角被全网黑"开挂"
→ 反转期待：证明自己的清白
→ 数据期待：被黑反而引发关注，在线人数暴涨
→ 排名期待：借势冲击月度榜单
→ 出圈期待：事件出圈，吸引大量路人关注
→ 财富期待：事件后收入暴涨
```

### 满足感分级详解（直播文）

|-----|-------|---------|---------|---------|

## 延迟满足法则（直播文）

### 黄金延迟公式

```
直播文延迟系数参考：
- 微期待（收到小礼物）：1章
```

### 直播文特有延迟技巧

| 黑粉节奏 | 被有组织地抹黑 | 舆论低谷，等待反转时机 |
| 榜单壁垒 | 头部主播垄断资源 | 榜单被老主播把控，等待突破时机 |
| 公会纠纷 | 合同/公会问题 | 被原公会打压，等待解约反击 |

## 直播文特有节奏：单场直播结构

### 一场直播的期待感设计

```
开场（5分钟）：抛钩子
├── 预告今天的"大事"
├── 念弹幕互动（微爽）
├── 制造悬念
└── 观众开始期待

发展（中期）：蓄势
├── 展示实力/创意内容
├── 数据逐渐攀升（小爽）
├── 出现挑战/竞争
└── 紧张感上升

高潮（后段）：爆发
├── 关键时刻来临
├── 数据暴涨（中爽~大爽）
├── 打脸/反转/破纪录
└── 观众弹幕沸腾

收尾（结尾）：留钩子
├── 感谢观众
├── 预告下次直播内容
└── 埋下新的期待种子
```

# 核心指令：直播文爽点节奏管理专家

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
*   项目初始化时设定爽点节奏
*   章节规划时分配爽点等级
*   审核直播剧情时检查爽点密度
*   发现爽点分布不均
*   设计单场直播的节奏结构

---

## 直播文爽点三级分类

### 小爽点（+1~+2）
**定义**：小数据涨、小互动、小打脸

**典型场景**：
- 收到第一个大额打赏
- 弹幕刷屏"666"
- 在线人数翻倍
- 打脸一条弹幕黑粉
- 粉丝数突破某个整数
- 榜单排名上升几名
- 成功完成一个直播挑战
- 被其他主播夸奖

**频率建议**：每1章一次

**作用**：维持快节奏爽感，让读者感受到持续进展

---

### 中爽点（+3~+4）
**定义**：阶段性数据突破、舆论小反转

**典型场景**：
- 在线人数突破万人大关
- 单场打赏收入破纪录
- 冲上周榜/月榜前十
- 舆论小反转（被澄清一个黑点）
- 成功连麦/击败一个大主播
- 粉丝团破万人
- 单个视频/切片出圈
- 签约平台或公会获得好条件

**频率建议**：每5章一次

**作用**：形成节奏高潮，给读者阶段性满足

---

### 大爽点（+5）
**定义**：全网级爆发、舆论大反转、登顶榜首

**典型场景**：
- 同时在线突破百万
- 登顶平台总榜第一
- 全网舆论大反转（从被全网黑到全网夸）
- 成为现象级话题/热搜第一
- 打脸最大的对手/黑粉头子
- 月收入突破百万
- 从无人知晓到全民皆知
- 商业帝国初具规模

**频率建议**：每20章一次

**作用**：全书记忆点，读者口碑传播点

---

## 爽点节奏配置（直播文推荐）

### 超高频节奏

| 节奏参数 | 配置 | 说明 |
|---------|------|------|
| 小爽间隔 | 1章 | 几乎每章都有数据增长或互动爽点 |
| 中爽间隔 | 5章 | 频繁的数据突破和打脸 |
| 大爽间隔 | 20章 | 每卷一次全网级大爆发 |

### 配置原则
1. **数据可见**：让读者看到具体的数字增长（在线人数、打赏金额、粉丝数）
2. **弹幕氛围**：用弹幕反馈来放大爽感（"牛逼"、"我来了"、"关注了"）
3. **节奏加速**：直播中的节奏比现实更快，压缩等待时间
4. **避免审美疲劳**：不能每章都是同样的数据暴涨模式，要变化爽点类型


# 核心指令：直播文世界观构建专家

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
*   需要构建直播文的平台生态和行业规则
*   需要设计公会体系和签约机制
*   需要规划主角的数据成长和商业路线
*   需要设计舆论场和粉丝文化
*   世界观设定出现矛盾或不符合现实逻辑

## 一、直播文世界观核心三要素

### 1. 平台生态
故事发生的直播平台环境和规则体系。

| 要素 | 说明 | 设计要点 |
|-----|------|---------|
| 平台定位 | 主流平台的类型和特色 | 游戏直播/短视频/综合直播，各有不同生态 |
| 推荐算法 | 流量分配的核心机制 | 算法偏好什么内容、如何获得推荐位 |
| 榜单体系 | 排名和曝光的关系 | 日榜/周榜/月榜/总榜，各榜单的权重 |
| 变现机制 | 主播赚钱的方式 | 打赏/广告/带货/签约费/品牌合作 |
| 违规规则 | 什么内容会被限流或封禁 | 避免踩线、被恶意举报的风险 |

### 2. 流量层级
直播平台的等级结构。

```
顶流层：平台一哥一姐、全网知名（主角的终极目标）
      ↓
头部层：平台前列、稳定高收入（中后期目标）
      ↓
腰部层：有稳定粉丝群、月入数万（中期目标）
      ↓
尾部层：几百到几千在线、收入不稳定（早期阶段）
      ↓
底层：几十人在线、几乎无收入（主角起点）
```

**层级设计原则**：
1. **可跨越**：每个层级之间都有突破路径
2. **有壁垒**：越往上越难突破（算法权重、资源垄断）
3. **有规律**：流量增长遵循一定的平台逻辑
4. **有偶然性**：偶尔会有"天选之子"一夜爆火

### 3. 商业体系
直播行业的利益链条。

| 角色 | 利益来源 | 与主角的关系 |
|-----|---------|-------------|
| 平台 | 抽成、广告、上市 | 提供舞台，也有规则束缚 |
| 公会 | 签约主播抽成、平台补贴 | 可以是助力也可以是牢笼 |
| MCN | 孵化网红、品牌对接 | 商业化加速器 |
| 品牌方 | 广告投放、带货合作 | 收入来源之一 |
| 粉丝 | 打赏、消费 | 核心收入来源 |

### 平台规则体系

```
【平台规则模板】
平台名称：（可虚构或使用真实平台）

├── 推荐机制
│   ├── 首页推荐：（算法推荐位）
│   ├── 分类榜单：（游戏/才艺/生活等）

│   ├── 粉丝团：（等级、特权、门槛）

└── 特殊机制
```

| 粉丝增量 | 单场新增粉丝数 | 成长速度指标 |
| 点赞数 | 内容认可度 | 小爽点来源 |

## 三、公会与势力体系

### 公会设定模板

```
【公会体系】
顶级公会
├── 资源：平台资源倾斜、独家推荐位
├── 主播：多位头部主播
├── 势力：能影响平台决策
└── 门槛：只签约有潜力的主播

中型公会
├── 资源：一定的平台资源
├── 主播：以腰部主播为主
├── 势力：在特定领域有影响力
└── 门槛：签约有一定条件

小型公会/工作室
├── 资源：有限
├── 主播：新人为主
├── 势力：几乎没有
└── 门槛：基本无门槛
```

### 公会与主角的关系设计

| 关系类型 | 说明 | 剧情作用 |
|---------|------|---------|
| 被打压 | 主角被大公会排挤 | 逆袭的反差铺垫 |
| 被挖角 | 公会用好条件挖主角 | 主角价值的体现 |
| 合作共赢 | 与公会互相成就 | 成长的助力 |
| 被背叛 | 公会不守承诺 | 冲突和转折的来源 |
| 自建公会 | 主角自己成立公会 | 成就感和权力感 |

## 四、舆论场与粉丝文化设定

### 舆论场设定

```
【舆论生态】
弹幕区 → 直播间即时舆论（最直接的反馈）
粉丝群 → 核心粉丝聚集地（忠诚度培养）
超话/贴吧 → 粉丝社区讨论（话题发酵）
微博/热搜 → 出圈舆论场（影响力放大）
短视频平台 → 切片二创传播（引流出圈）
```

### 粉丝群体设计

| 粉丝类型 | 特征 | 行为模式 | 价值 |
|---------|------|---------|------|
| 榜一大哥 | 高消费、高粘性 | 持续打赏、维护主播 | 核心收入来源 |
| 铁粉/老粉 | 高忠诚、中消费 | 每天来看、自发安利 | 稳定基本盘 |
| 路人粉 | 低忠诚、低消费 | 偶尔来看看 | 潜在增长点 |
| 黑粉 | 负面关注 | 找茬、带节奏 | 反向流量/打脸素材 |
| CP粉 | 情感投射 | 磕主播相关CP | 话题制造机 |

### 舆论反转设计模板

```
【舆论反转公式】
被黑阶段（低谷）：
├── 起因：（被误解/被断章取义/被恶意剪辑）
├── 发酵：（黑粉带节奏/竞争对手推波助澜）
├── 低谷：（全网骂/掉粉/被抵制）
└── 铺垫线索：（暗中埋下真相的伏笔）

反转阶段（高潮）：
├── 契机：（新证据/当事人澄清/第三方作证）
├── 反转：（舆论开始转向）
├── 高潮：（全网道歉/黑粉打脸/粉丝暴涨）
└── 后续：（新的机遇/更高起点）
```

## 五、收入体系设计

### 主播收入阶段

| 阶段 | 月收入 | 典型状态 | 社会认知 |
|-----|-------|---------|---------|
| 底层期 | 0-1000元 | 没人看、自娱自乐 | "不务正业" |
| 起步期 | 1000-10000元 | 有小批粉丝、偶尔有打赏 | "能赚点零花钱" |
| 发展期 | 1万-10万元 | 稳定粉丝群、有固定打赏 | "靠这个能活了" |
| 头部期 | 10万-100万元 | 平台知名、品牌合作 | "赚得比上班多" |
| 顶流期 | 100万+ | 全网知名、商业帝国 | "人生赢家" |

### 收入来源合理性检查

- [ ] 打赏金额是否与在线人数匹配？
- [ ] 品牌合作是否符合主播定位？
- [ ] 收入增长速度是否有逻辑支撑？
- [ ] 是否考虑了税收、平台抽成等现实因素？
- [ ] 是否有收入波动的合理性？（不能每月匀速增长）
