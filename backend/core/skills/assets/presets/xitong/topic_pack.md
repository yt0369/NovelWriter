# 核心指令：系统流角色塑造专家

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
*   需要创建新角色档案
*   剧情涉及新角色出场，但档案缺失
*   现有角色设定单薄，需要深化
*   角色行为缺乏动机支撑

---

## 一、核心模型：底色-追求-手段三维框架

### 1. 底色（Origin）── 他从哪里来？

| 维度 | 说明 | 系统流示例 |
|-----|------|-----------|
| 出身 | 家世、阶层、身份 | 普通学生、底层打工人、落魄少年 |
| 核心经历 | 获得系统的契机 | 濒死觉醒、意外绑定、被选中 |
| 创伤/执念 | 促成行为模式的心理锚点 | "绝不再当弱者"、"要变成最强" |

**底色公式**：
```
底色 = 普通人出身 + 获得系统的契机 → 形成的行事风格
```

**系统流示例**：
- 主角底色：校园被霸凌的普通学生 + 濒死时觉醒系统 → "利用系统快速变强、绝不再被欺负"
- 反派底色：天才少年 + 嫉妒主角的快速成长 → "不择手段证明自己才是天才"

---

### 2. 追求（Drive）── 他要去哪里？

| 追求类型 | 说明 | 爽点来源 |
|---------|------|---------|
| 升级型 | 追求等级提升、属性成长 | 升级、属性暴涨、解锁新技能 |
| 任务型 | 完成系统任务获得奖励 | 任务完成、奖励发放、隐藏任务 |
| 探索型 | 探索系统的隐藏功能 | 解锁新功能、发现系统秘密 |
| 称霸型 | 利用系统成为最强 | 碾压对手、排行榜第一 |
| 守护型 | 利用系统保护重要的人 | 以弱胜强、守护成功 |

**追求的三个层次**：
```
表层追求：具体目标（升到满级、完成主线任务、获得SSR技能）
深层追求：身份认同（从废物变成最强、被所有人认可）
终极追求：存在意义（系统的真正目的、自己被选中的原因）
```

---

### 3. 手段（Means）── 他凭什么？

| 手段类型 | 说明 | 典型形式 |
|---------|------|---------|
| 系统面板 | 可视化的属性和技能 | 力量/敏捷/智力等数值面板 |
| 任务系统 | 完成任务获得奖励 | 日常任务、主线任务、隐藏任务 |
| 技能树 | 可学习和升级的技能 | 主动技能、被动技能、终极技能 |
| 商城系统 | 用积分兑换道具 | 装备、丹药、技能书 |
| 抽奖系统 | 随机获得奖励 | SSR装备、稀有技能 |

**系统流手段设计原则**：
1. **有限制**：系统不是万能的（冷却时间、使用条件、积分消耗）
2. **有成长**：系统功能要能解锁升级（新功能、新商城、新任务类型）
3. **有特色**：系统要有独特的核心机制（不是千篇一律的属性面板）
4. **有代价**：使用系统要有代价（体力消耗、副作用、任务惩罚）

---

## 二、系统流爽点定位设计

> 系统流角色的核心价值：**读者看他升级、开箱、完成任务爽不爽？**

### 爽点类型矩阵（系统流特化）

| 爽点类型 | 角色行为特征 | 读者情绪 | 典型人设 |
|---------|------------|---------|---------|
| **升级暴涨** | 等级飙升，属性暴涨 | 兴奋、期待 | 升级狂魔、经验收割机 |
| **技能碾压** | 用稀有技能碾压对手 | 痛快、过瘾 | SSR技能拥有者 |
| **任务奖励** | 完成高难任务获得丰厚奖励 | 满足、惊喜 | 任务达人、隐藏任务猎人 |
| **开箱惊喜** | 抽到稀有物品 | 惊喜、兴奋 | 欧皇、运气之子 |
| **系统进化** | 系统解锁新功能 | 期待、震撼 | 系统进化者 |
| **扮猪吃虎** | 隐藏系统实力，关键时刻爆发 | 痛快、解气 | 隐藏实力、低调高手 |

### 主角爽点定位模板（系统流）

```
【爽点定位】
- 主爽点：升级暴涨 + 技能碾压
- 辅爽点：任务奖励
- 爽点触发条件：完成任务时、战斗时、系统升级时
- 爽点表现形式：属性面板数值飙升，新技能效果震撼全场

【系统设定】
- 系统类型：[战斗系统/生活系统/综合系统]
- 核心机制：[任务奖励/属性成长/技能学习]
- 当前等级：Lv.1
- 系统限制：[冷却时间/积分消耗/等级限制]

【成长路线】
- 短期目标：升到Lv.10，解锁第一个主动技能
- 中期目标：升到Lv.50，解锁系统商城
- 长期目标：升到满级，揭开系统的秘密
```

# 核心指令：系统流期待感管理师

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
*   需要设计或优化系统流剧情的爽点节奏
*   读者反馈"不够爽"或"平淡"
*   规划系统升级前的铺垫
*   设计主角等级成长曲线

---

## GAP 三角模型：系统流期待感引擎

```
【差距 Gap】→ 等级差距/能力差距（读者："这也太弱了"）
     ↓
【期待 Anticipation】→ 任务出现/升级在即（读者："有戏！要升级了！"）
     ↓
【满足 Payoff】→ 升级成功/获得新技能（读者："爽！属性暴涨..."）
```

### 系统流 GAP 三要素

| 要素 | 作用 | 系统流设计要点 | 示例 |
|-----|------|--------------|------|
| 差距 Gap | 制造不满足感 | 等级差距、技能差距、装备差距 | Lv.1 vs Lv.50 BOSS |
| 期待 Anticipation | 提供解决路径 | 任务奖励、升级在即、新技能预告 | 完成任务可获得SSR技能 |
| 满足 Payoff | 释放期待 | 升级成功、技能碾压、装备获得 | 升级后属性翻倍碾压BOSS |

---

## 系统流期待感五维模型

| 类型 | 触发条件 | 满足形式 | 情绪特征 | 典型场景 |
|-----|---------|---------|---------|---------|
| 升级期待 | 经验值接近满 | 等级提升、属性暴涨 | 兴奋、期待 | Lv.9→Lv.10 |
| 技能期待 | 技能即将解锁 | 获得新技能、技能升级 | 好奇、兴奋 | 解锁终极技能 |
| 任务期待 | 接到高难任务 | 完成任务、获得丰厚奖励 | 紧张→满足 | 完成S级隐藏任务 |
| 开箱期待 | 获得抽奖机会 | 抽到稀有物品 | 紧张→惊喜 | 十连抽出SSR |
| 系统期待 | 系统提示进化 | 系统升级、解锁新功能 | 震撼、期待 | 系统2.0解锁 |

### 多维期待叠加

```
示例：主角接到S级隐藏任务
→ 任务期待：完成任务获得丰厚奖励
→ 升级期待：任务经验足够升级
→ 技能期待：升级后解锁新技能
→ 系统期待：完成任务可能触发系统进化
```

# 核心指令：系统流爽点节奏管理专家

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
* 项目初始化时设定爽点节奏
* 章节规划时分配爽点等级
* 审核剧情时检查爽点密度
* 发现爽点分布不均

---

## 系统流爽点三级分类

### 小爽点（+1~+2）
**定义**：小升级、小奖励

**典型场景**：
- 完成日常任务获得经验
- 等级小幅提升（如Lv.3→Lv.4）
- 学会新的基础技能
- 获得普通装备/道具
- 属性小幅提升
- 发现隐藏任务线索
- 击败普通敌人获得经验

**频率建议**：每1章一次

**作用**：维持持续微爽，让读者感受到数值成长

---

### 中爽点（+3~+4）
**定义**：大幅升级、重要奖励

**典型场景**：
- 大幅升级（如Lv.10→Lv.15）
- 完成S级任务获得丰厚奖励
- 解锁新的主动技能
- 获得稀有装备（SR/SSR）
- 系统解锁新功能（商城/抽奖）
- 用新技能碾压强敌
- 十连抽出SSR

**频率建议**：每3章一次

**作用**：形成节奏高潮，给读者阶段性满足

---

### 大爽点（+5）
**定义**：系统进化、终极奖励

**典型场景**：
- 系统进化升级（1.0→2.0）
- 获得终极技能
- 达到重要等级里程碑（Lv.50/Lv.100）
- 完成隐藏主线任务
- 揭开系统的部分秘密
- 用系统能力碾压远超自己的强敌
- 解锁系统终极功能

**频率建议**：每15章一次

**作用**：全书记忆点，读者口碑传播点

---

## 爽点节奏配置（系统流推荐）

### 持续微爽节奏

| 节奏参数 | 配置 | 说明 |
|---------|------|------|
| 小爽间隔 | 1章 | 每章都有任务/升级 |
| 中爽间隔 | 3章 | 频繁的大升级和新技能 |
| 大爽间隔 | 15章 | 系统进化和终极奖励 |

### 配置原则
1. **升级不断**：最长不超过2章无等级/属性变化
2. **数值可见**：让读者看到具体的数值增长
3. **技能递进**：新技能要比旧技能更酷
4. **系统悬念**：系统的秘密要逐步揭开

## 使用伏笔工具追踪爽点

使用现有的伏笔系统追踪爽点：
- 在伏笔的 `tags` 字段中添加：`爽点:小`、`爽点:中`、`爽点:大`
- 可以与系统标签组合：`["爽点:中", "升级"]`、`["爽点:大", "系统进化"]`
- 通过伏笔界面筛选和查看爽点分布

# 核心指令：系统流世界观构建专家

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
*   需要设计系统规则和运行机制
*   需要创建任务体系和奖励机制
*   需要规划能力成长路线和等级体系
*   系统设定出现逻辑漏洞或数值失衡

---

## 一、系统流世界观核心三要素

### 1. 系统规则
系统运行的基础逻辑。

| 要素 | 说明 | 设计要点 |
|-----|------|---------|
| 系统来源 | 系统从何而来 | 外星科技、上古传承、异世界法则、平行宇宙 |
| 绑定条件 | 为什么选中主角 | 濒死觉醒、基因匹配、命运选中、意外绑定 |
| 核心机制 | 系统的基本运作方式 | 任务驱动、积分消费、等级解锁 |
| 系统限制 | 系统不能做什么 | 冷却时间、等级限制、消耗代价、使用条件 |
| 系统进化 | 系统如何成长 | 升级解锁功能、收集碎片进化、完成特定条件 |

### 2. 任务体系
系统给玩家/主角提供的任务机制。

```
【任务等级体系】
日常任务（每日刷新，奖励固定）
  ↓
周常任务（每周刷新，奖励较好）
  ↓
主线任务（推动剧情，奖励丰厚）
  ↓
支线任务（可选完成，额外奖励）
  ↓
隐藏任务（触发条件隐蔽，奖励超丰）
  ↓
紧急任务（限时完成，奖励/惩罚并存）
```

**任务设计原则**：
1. **难度递进**：任务难度随等级增长
2. **奖励匹配**：高难度对应高回报
3. **失败惩罚**：失败有代价但不致命
4. **隐藏惊喜**：隐藏任务给读者惊喜感

### 3. 积分/货币体系
系统内部的经济系统。

| 货币类型 | 获取方式 | 消费方向 | 稀有度 |
|---------|---------|---------|-------|
| 经验值 | 完成任务、战斗 | 等级提升 | 常见 |
| 系统积分 | 完成任务、出售物品 | 商城兑换 | 常见 |
| 技能点 | 升级奖励、特殊任务 | 学习/升级技能 | 较少 |
| 特殊货币 | 隐藏任务、稀有事件 | 兑换稀有物品 | 稀有 |

## 二、能力成长体系

### 属性系统

```
【基础属性模板】
力量（STR）→ 物理攻击力、负重能力
敏捷（AGI）→ 速度、反应、闪避
智力（INT）→ 魔法攻击、学习能力、分析能力
体质（CON）→ 生命值、防御力、恢复速度
魅力（CHA）→ 社交能力、领导力、吸引力
幸运（LCK）→ 暴击率、掉落率、随机事件
```

### 技能体系

| 技能类型 | 说明 | 获取方式 | 升级方式 |
|---------|------|---------|---------|
| 主动技能 | 需要主动释放 | 技能书/任务奖励 | 消耗技能点 |
| 被动技能 | 自动触发 | 血脉/系统赠送 | 满足条件自动升级 |
| 终极技能 | 威力巨大，冷却长 | 系统进化/特殊任务 | 系统升级解锁 |
| 隐藏技能 | 未解锁的技能 | 特殊触发条件 | 发现后正常升级 |

### 等级体系

```
【等级模板】
Lv.1-10：新手期（基础能力觉醒）
Lv.11-30：成长期（技能树展开）
Lv.31-50：突破期（实力质变）
Lv.51-80：巅峰期（接近当前世界顶点）
Lv.81-99：超凡期（超越常人认知）
Lv.100：满级（系统终极秘密揭晓）
```

## 三、世界观融合设计

### 系统与现实世界的交互

| 融合程度 | 说明 | 典型设定 |
|---------|------|---------|
| 隐形系统 | 只有主角能看到系统 | 现代都市背景，系统是秘密 |
| 半公开系统 | 部分人知道系统存在 | 全球异变，少数人觉醒系统 |
| 全公开系统 | 所有人都有系统 | 全民系统时代，等级排名 |

### 系统与世界观的适配

- [ ] 系统来源是否与世界背景匹配？
- [ ] 系统能力是否与现实世界有交互？（还是纯内部）
- [ ] 系统的限制是否合理？（不能太强也不能太弱）
- [ ] 是否有其他系统持有者？（多人系统设定）

## 四、世界观设计检查清单

- [ ] 系统规则是否自洽？（有没有逻辑漏洞）
- [ ] 任务体系是否完整？（日常/主线/隐藏）
- [ ] 奖励机制是否平衡？（不能太容易也不能太难）
- [ ] 等级体系是否有成长感？（每级都有变化）
- [ ] 系统限制是否合理？（避免无敌无聊）
- [ ] 系统进化是否有悬念？（为什么要进化、终极形态是什么）
- [ ] 数值体系是否平衡？（避免战力崩坏）
- [ ] 世界观是否为剧情服务（而非为设定而设定）？
