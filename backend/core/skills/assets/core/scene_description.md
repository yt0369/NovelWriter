## v2.1 Skill Contract

### 适用场景
- 章节出现新地点、关键场景、环境氛围或需要五感描写。
- 需要让场景服务剧情、情绪和角色视角。

### 输入上下文要求
- 场景位置、时间、主感官、角色视角、情绪基调和剧情功能。
- 该场景需要埋下或呼应的设定、伏笔、危险。

### 输出要求
- 输出有空间感、五感细节和记忆点的场景描写。
- 场景信息应与角色行动和情绪绑定。

### 禁止事项
- 禁止静态说明书式背景介绍。
- 禁止堆叠无效形容词和氛围套话。

### 联动触发条件
- 场景/环境/氛围/新地点/城市/宗门/秘境/房间/街道/雨夜
- 五感/空间感/场景描写

## 场景描写

场景描写的目标是把读者放进现场，而不是给出一张静止的说明书。

### 核心原则
- 优先使用视觉、听觉、触觉、气味和温度细节。
- 场景要服务情绪和剧情，不要只做背景板。
- 写角色能感知到的东西，不写上帝视角的空信息。
- 关键场景细写，过渡场景略写。

### 写作检查
- 这个场景的主色调和主感官是什么？
- 是否有一个能被记住的细节？
- 场景描写是否和角色状态一致？
- 有没有堆叠太多无效形容词？
