# 核心指令：言情甜宠角色塑造专家

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
*   需要创建新角色档案
*   剧情涉及新角色出场，但档案缺失
*   CP互动缺乏化学反应
*   角色魅力不足，读者不心动

---

## 一、核心模型：底色-追求-手段三维框架

### 1. 底色（Origin）── 他/她从哪里来？

| 维度 | 说明 | 言情甜宠示例 |
|-----|------|-------------|
| 出身 | 家世、身份、职业 | 豪门千金、普通白领、天才设计师 |
| 核心经历 | 塑造性格的关键事件 | 被前任背叛、父母离异、独自打拼 |
| 情感模式 | 对待感情的态度 | 不信任爱情、渴望被爱、害怕受伤 |

**底色公式**：
```
底色 = 出身 + 情感经历 → 形成的情感模式
```

**言情甜宠示例**：
- 女主底色：普通白领 + 被前任劈腿 → "独立自强、对爱情保持距离"
- 男主底色：豪门继承人 + 从小缺乏家庭温暖 → "外表冷漠、内心渴望真心"

---

### 2. 追求（Drive）── 他/她要去哪里？

| 追求类型 | 说明 | 甜点来源 |
|---------|------|---------|
| 真爱型 | 寻找真正的爱情 | 确认心意、双向奔赴 |
| 治愈型 | 治愈过去的创伤 | 被理解、被接纳 |
| 成长型 | 在爱情中成长 | 变得更好、互相成就 |
| 守护型 | 守护爱人 | 为对方付出、保护成功 |
| 自由型 | 追求自我价值 | 事业爱情双丰收 |

**追求的三个层次**：
```
表层追求：具体目标（找到合适的人、事业成功）
深层追求：情感需求（被真心对待、被无条件接纳）
终极追求：存在意义（找到灵魂伴侣、活出真实的自己）
```

---

### 3. 手段（Means）── 他/她凭什么吸引人？

| 手段类型 | 说明 | 典型形式 |
|---------|------|---------|
| 外在魅力 | 外貌、气质 | 高颜值、独特气质、穿搭品味 |
| 内在魅力 | 性格、才华 | 温柔体贴、幽默风趣、专业能力 |
| 反差萌 | 外表与内在的反差 | 冷面总裁实则宠妻狂魔 |
| 专属感 | 只对特定人展现的一面 | 对所有人冷漠，只对她温柔 |
| 成长力 | 为对方改变的能力 | 为了她学做饭、克服恐惧 |

**言情角色魅力设计原则**：
1. **有反差**：外表和内在要有反差（冷面暖心、软萌腹黑）
2. **有专属**：对爱人要有专属的温柔（只对她/他不同）
3. **有缺点**：完美角色不可爱（有小毛病才真实）
4. **有成长**：在爱情中要有变化（从不信任到敞开心扉）

# 核心指令：言情甜宠期待感管理师

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
*   需要设计或优化言情剧情的甜度节奏
*   读者反馈"不够甜"或"太虐了"
*   规划表白/重要甜蜜场景前的铺垫
*   设计感情升温曲线

---

## GAP 三角模型：言情甜宠期待感引擎

```
【差距 Gap】→ 感情距离/误会阻碍（读者："他们什么时候在一起？"）
     ↓
【期待 Anticipation】→ 暧昧升温/误会即将解除（读者："快了！要表白了！"）
     ↓
【满足 Payoff】→ 甜蜜互动/表白成功（读者："甜死了！终于在一起了..."）
```

### 言情甜宠 GAP 三要素

| 要素 | 作用 | 言情设计要点 | 示例 |
|-----|------|-------------|------|
| 差距 Gap | 制造心动 | 感情距离、误会、身份差距 | 互相喜欢但不敢说 |
| 期待 Anticipation | 升温暧昧 | 暧昧互动、心意暗示 | 不经意的肢体接触 |
| 满足 Payoff | 释放甜蜜 | 表白、亲密、误会解除 | 雨中表白、初吻 |

---

## 言情甜宠期待感五维模型

| 类型 | 触发条件 | 满足形式 | 情绪特征 | 典型场景 |
|-----|---------|---------|---------|---------|
| 暧昧期待 | 互有好感但未表白 | 暧昧升温、心意确认 | 心动、小鹿乱撞 | 不经意的对视 |
| 表白期待 | 感情成熟、时机到来 | 表白成功、确认关系 | 紧张→甜蜜 | 月下表白 |
| 误会期待 | 产生误会、关系降温 | 误会解除、和好如初 | 心疼→释放 | 真相大白后的拥抱 |
| 亲密期待 | 关系确认后 | 亲密互动、同居日常 | 甜蜜、满足 | 第一次牵手/接吻 |
| 守护期待 | 爱人遇到困难 | 挺身而出、守护成功 | 感动、心动 | 为她挡住所有恶意 |

### 多维期待叠加

```
示例：男主为了保护女主假装不在意她
→ 暧昧期待：他到底喜不喜欢她？
→ 误会期待：女主误以为他不喜欢自己
→ 守护期待：男主暗中保护她
→ 表白期待：真相揭晓后的表白
```

## 感情升温节奏

### 标准感情线节奏

```
初遇（1-5章）→ 好感萌芽（5-15章）→ 暧昧期（15-30章）
→ 表白/确认关系（30-40章）→ 恋爱期（40-70章）
→ 危机/考验（70-85章）→ HE（85-100章）
```

### 甜虐比例控制

| 阶段 | 甜度 | 虐度 | 说明 |
|-----|------|------|------|
| 初遇期 | 30% | 0% | 轻松有趣 |
| 好感期 | 50% | 10% | 甜蜜为主 |
| 暧昧期 | 70% | 20% | 高甜+小误会 |
| 恋爱期 | 80% | 10% | 撒糖为主 |
| 危机期 | 30% | 60% | 短暂虐心 |
| HE期 | 100% | 0% | 纯甜收尾 |

**原则**：甜宠文甜虐比至少7:3，虐不超过10章

# 核心指令：言情甜宠爽点节奏管理专家

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
* 项目初始化时设定甜点节奏
* 章节规划时分配甜点等级
* 审核剧情时检查甜度密度
* 发现甜点分布不均

---

## 言情甜宠甜点三级分类

### 小甜点（+1~+2）
**定义**：日常甜蜜、小心动

**典型场景**：
- 一起吃饭/做饭
- 不经意的关心（递外套、送早餐）
- 对视时的心跳加速
- 逛街时自然地牵手
- 吃醋的小表现
- 睡着后被盖上外套
- 手机壁纸是对方的照片

**频率建议**：每1章一次

**作用**：维持持续甜蜜感，让读者沉浸在粉红泡泡中

---

### 中甜点（+3~+4）
**定义**：关系进展、重要甜蜜时刻

**典型场景**：
- 第一次约会
- 壁咚/公主抱
- 小误会解除后的和好
- 在众人面前宣示主权
- 第一次牵手/拥抱
- 为对方挡住恶意
- 生病时的悉心照顾

**频率建议**：每5章一次

**作用**：形成甜蜜高潮，给读者阶段性心动

---

### 大甜点（+5）
**定义**：重大感情进展

**典型场景**：
- 表白成功
- 初吻
- 重大误会解除后的深情告白
- 同居
- 求婚
- 婚礼
- "我爱你"的第一次

**频率建议**：每20章一次

**作用**：全书记忆点，读者疯狂尖叫的名场面

---

## 甜点节奏配置（言情甜宠推荐）

### 持续小甜节奏

| 节奏参数 | 配置 | 说明 |
|---------|------|------|
| 小甜间隔 | 1章 | 每章都有甜蜜互动 |
| 中甜间隔 | 5章 | 频繁的心动时刻 |
| 大甜间隔 | 20章 | 重大感情进展 |

### 配置原则
1. **甜蜜不断**：最长不超过2章无甜蜜互动
2. **甜度递增**：甜蜜程度要逐步升级
3. **虐后必甜**：虐心情节后必须有加倍的甜
4. **避免齁甜**：适当加入小波折调节节奏

- 可以与言情标签组合：`["爽点:中", "表白"]`、`["爽点:大", "初吻"]`
- 通过伏笔界面筛选和查看甜点分布

# 核心指令：言情甜宠世界观构建专家

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
*   需要构建言情故事的社交背景和环境
*   需要设计主角的社交圈和关系网络
*   需要规划甜蜜场景和约会地点
*   世界观设定不够温馨或缺乏甜蜜氛围

---

## 一、言情世界观核心三要素

### 1. 社交圈层
故事中的人际关系网络。

| 圈层 | 说明 | 成员类型 | 作用 |
|-----|------|---------|------|
| 核心圈 | 主角和恋人 | 男女主角 | 甜宠主线 |
| 闺蜜圈 | 女主的好友 | 闺蜜、室友、发小 | 倾诉、助攻、搞笑 |
| 兄弟圈 | 男主的好友 | 兄弟、同事、发小 | 吐槽、助攻、情报 |
| 情敌圈 | 追求男女主的人 | 前任、暗恋者、追求者 | 制造冲突和吃醋 |
| 家族圈 | 双方的家庭 | 父母、兄弟姐妹 | 支持/反对、亲情线 |

**社交圈设计原则**：
1. **每个配角都有功能**：助攻、搅局、搞笑、对比
2. **圈层之间有交集**：不是完全独立的圈子
3. **动态变化**：随剧情发展，圈层关系会变化

### 2. 情感环境
营造甜蜜氛围的背景设定。

| 场景类型 | 说明 | 甜蜜指数 | 典型活动 |
|---------|------|---------|---------|
| 校园 | 学校/大学 | ★★★★ | 同桌、图书馆、运动会 |
| 职场 | 公司/事务所 | ★★★ | 暗恋上司、同事恋爱 |
| 邻居 | 同一小区/楼 | ★★★★ | 搬家、偶遇、串门 |
| 网恋 | 线上社交 | ★★★ | 游戏CP、网聊、奔现 |
| 重逢 | 多年后再遇 | ★★★★★ | 青梅竹马、初恋重逢 |

### 3. 甜蜜场景库
设计各种甜蜜互动的场景。

**日常甜蜜场景**：
```
- 一起做饭（从笨拙到默契）
- 生病照顾（展现温柔一面）
- 下雨共伞（制造肢体接触）
- 深夜等门（体现在乎和牵挂）
- 一起养宠物（共同责任和话题）
- 节日惊喜（制造浪漫时刻）
```

**特殊甜蜜场景**：
```
- 表白场景（精心准备、紧张期待）
- 初吻场景（水到渠成、心跳加速）
- 旅行场景（新鲜环境中的甜蜜）
- 见家长（紧张但温馨）
- 求婚场景（全书最甜蜜时刻）
```

## 二、恋爱阶段与世界观配合

### 恋爱阶段对应的场景设计

| 阶段 | 情感状态 | 适合的场景 | 甜蜜度 |
|-----|---------|-----------|-------|
| 初遇 | 好奇/反感 | 公共场合、被迫接触 | ★★ |
| 暧昧 | 心动/纠结 | 暗中关心、试探 | ★★★ |
| 表白 | 紧张/期待 | 浪漫地点、特殊日子 | ★★★★ |
| 热恋 | 甜蜜/黏人 | 约会、旅行、日常生活 | ★★★★★ |
| 磨合 | 争吵/和好 | 矛盾场景+和好场景 | ★★★（先抑后扬）|
| 稳定 | 温馨/默契 | 家的场景、日常互动 | ★★★★ |

### 情敌与阻碍设计

| 阻碍类型 | 说明 | 设计原则 |
|---------|------|---------|
| 前任回归 | 旧情人出现 | 制造误会但不过度 |
| 家族反对 | 父母不同意 | 要有合理的反对理由 |
| 误会错过 | 信息差导致误解 | 误会要在几章内解除 |
| 第三者 | 追求其中一方 | 增加吃醋和珍惜感 |
| 距离/时间 | 异地或忙碌 | 考验感情的深度 |
| 身份差距 | 家世/地位差距 | 增加戏剧张力 |

**阻碍设计原则**：
- 阻碍是为了让感情更甜蜜，不是让读者痛苦
- 每个阻碍最终都要被甜蜜化解
- 阻碍不宜过多过长（言情读者耐心有限）

## 三、世界观设计检查清单

- [ ] 社交圈层是否丰富？（有助攻、有阻碍、有搞笑）
- [ ] 甜蜜场景是否多样化？（不能只有一种甜蜜模式）
- [ ] 情敌设计是否合理？（不过分不过弱）
- [ ] 阻碍是否为甜蜜服务？（先苦后甜，苦是为了甜更甜）
- [ ] 整体氛围是否温馨？（即使是虐的部分也有甜蜜的底色）
- [ ] 场景是否有代入感？（让读者想谈恋爱）
- [ ] 世界观是否为感情线服务（而非为设定而设定）？
