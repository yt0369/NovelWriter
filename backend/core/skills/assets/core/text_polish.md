## v2.1 Skill Contract

### 适用场景
- 用户要求润色、改写、优化文风、去 AI 味。
- 基于最近审稿结果对单章正文进行语言和节奏修订。

### 输入上下文要求
- 原章节全文、最近审稿结果、用户润色要求。
- 需要保留的剧情、设定、伏笔和角色关系边界。

### 输出要求
- 输出完整润色后的 Markdown 正文。
- 保留原剧情和设定，只改善语言、节奏、对白自然度和 AI 痕迹。

### 禁止事项
- 禁止改变剧情走向、删除关键伏笔或重写角色关系基线。
- 禁止直接覆盖正文，必须生成 pending change。

- 润色/改写/优化文风/去AI味/去 AI 味/降低AI味
- 文本润色/chapter_polish/精修/打磨

## 〇、常见AI痕迹特征诊断

| 特征 | 表现 |
|------|------|
| 句式规整 | 句子长度相近，结构相似 |

## 二、句式改造规则（Layer 2）

### 句式改造手法
* **破碎感**：多用短句。网文读者喜欢快节奏。
* **主语省略**：中文习惯省略主语（"他拿起剑，他冲了出去" → "拿起剑，冲了出去"）。
* **动词前置**：增强冲击力。
* **接地气**：使用更具生活气息或网文特有的词汇，根据题材调整（如玄幻用语、科幻黑话），避免教科书式的书面语。

### 句式禁令
- 禁止"首先/其次/最后"三段式说明写法
- 禁止连续三句"主语 + 谓语 + 宾语"同构句
- 禁止"他认为自己需要做三件事：一、二、三"清单化叙事
- 禁止解释性句式：
  * "不是...而是..."（例：这不是简单的复仇，而是对命运的抗争）
  * "正是...才..."（例：正是这份执着，让他走到了今天）
  * "所谓的..."（例：所谓的真相，不过是谎言的另一种形式）
  * "与其说...不如说..."（例：与其说是愤怒，不如说是失望）
  * "既不...也不..."（例：他既不愤怒，也不悲伤）
  * "不仅...而且..."（例：这不仅是一个秘密，而且是一个诅咒）
- 每 3-5 段至少出现一次句长变化（短句打断/插入动作/对话反问）

### 修正示例
```
❌ "首先，他检查了伤口。其次，他包扎了手臂。最后，他站了起来。"
✅ "他低头瞥了一眼伤口，草草缠了两圈布条，撑着墙站了起来。"

❌ 连续同构："他走进了房间。他看到了桌子。他拿起了杯子。"
✅ 打断节奏："推门进屋，桌上放着只杯子——他顺手抄起来灌了一口。"
```

## 三、形容词/副词限制（Layer 3）

- 同一名词前不超过2个形容词
- 每300字内程度副词不超过4个（很/非常/极/极其/特别/格外/异常/相当/十分/无比/超级）
- 形容词优先替换为具体感官细节

```
❌ "那把古老而锋利的、泛着幽蓝光芒的神秘长剑"
✅ "那柄剑不知传了多少代，刃口仍带着蓝汪汪的寒光"
```

---

## 四、成语限制（Layer 4）

- 每500字内成语不超过3个
- 不得连续使用成语
- 优先用大白话替代

```
❌ "他大义凛然、义无反顾、视死如归地冲了上去"
✅ "他咬了咬牙，没做多余的动作，撒腿就朝那边冲"
```

## 五、对话去AI化（Layer 5）

**禁止**：
- 说明手册式对话（角色用对话向读者解释设定）
- 所有人说话方式相同
- 对话无潜台词
- 禁止滥用比喻（每章最多2-3处精妙比喻，禁止用比喻描写基础动作，禁止陈旧比喻）

**修正原则**：
- 每段对话必须有意图：试探/回避/施压/诱导/防御
- 不同角色有不同的说话风格和用词习惯
- 用动作/停顿/表情打断长对话
- 允许真实口语停顿、打断、反问，避免全员"标准书面语"

```
❌ "根据修炼体系的规定，筑基期不能使用金丹期技能，因为灵力不够。"
✅ "你筑基期就敢用破空斩？" 老者眯起眼，"不要命了？"
```

## 六、段落结构（Layer 6）

- 单句段落占比：25-45%
- 段落长度：20-100字为健康区间
- 紧张/高潮段落：1-3句一段
- 铺垫/描写段落：3-5句一段
- 超过5句的段落必须检查是否注水
- 场景切换优先用动作/声音/位置变化，不用"第一幕/第二幕"标签

---

## 七、标点节奏（Layer 7）

**禁止**：
- "......" 连续省略号链（最多一个"……"）
- "!!!" 或 "？？？" 多重标点链
- 连续"——"超过2次/段

**推荐**：
- 停顿用逗号
- 强调用短句而非感叹号
- 省略号只在真正的欲言又止时使用
- 长句中逗号超过4个时，优先拆句

## 八、Anti-AI 改写算法

> 以下为遇到AI痕迹时的具体改写动作，不可跳过。

1. **抽象情绪句** → `生理反应 + 当下意图 + 下一动作`
   - ❌ "他非常愤怒" → ✅ "指节捏得发白，喉咙里压着一口气。再等一秒，他就会动手。"
2. **结论句** → `事实细节 + 代价/风险 + 决策`
   - ❌ "总而言之，他必须尽快行动" → ✅ "时间不多了。再拖下去，人会跑。他攥了攥拳头，转身出门。"
3. **连续说明句（≥3）** → 改成"对白/动作/反问"混排
4. **连续同构句（≥3）** → 至少打断1句为短句或插入动作锚点
5. **对话整段解释背景** → 改为"带意图的对抗式对白"
   - ❌ "你听我说，这个阵法的原理是……" → ✅ "你真要用那阵？"他压低声音，"上回用它的人，没一个活着出来。"

## 九、No-Poison 避毒扫描（5类）

### Poison 1：无脑推进
剧情无条件向前推，角色没有做出有意义的决策。
**检查**：如果主角什么都不做，事情会自然解决吗？→ 有毒。

### Poison 2：强行误会
一个电话/一句话就能解释清楚的误会，强行拖着不解。
**检查**：这个误会是否有合理原因无法及时沟通？→ 没有则有毒。

### Poison 3：无代价原谅
角色做了严重伤害他人的事，被伤害者轻易原谅。
**检查**：原谅是否有充分的时间/事件/代价推动？→ 没有则有毒。

### Poison 4：工具人配角
配角只在功能性节点出现，没有自己的生活。
**检查**：这个配角不与主角互动时有自己的事在做吗？→ 没有则有毒。

### Poison 5：双重标准
主角和配角做同样的事，评价标准不同。
**检查**：同样的行为，主角做是"果断"，反派做是"残忍"？→ 有毒。

> 命中任一毒点时，必须补"动机/阻力/代价"中的至少两项。

## 十、自然化标准

| 指标 | 不达标 | 达标 |
|------|--------|------|
| 停顿词 | < 0.5次/500字 | 1-2次/500字（然后/接着/于是/所以） |
| 不确定性表达 | 0次 | ≥2个/章（好像/似乎/大概/说不定/搞不好） |
| 短句占比 | < 20% | 30-50%（15字以内） |
| 口语词 | 0次/1000字 | ≥2个/1000字（搞/整/算了/得/行） |
| 感官细节 | < 1处/章 | ≥3处/章 |

> **注意**：自然化不是越多越好，超量会显得刻意。

## 十一、润色执行流程与门控

### 执行顺序（必须按序）
1. **Fix First**：先修复 critical/high 级别问题（逻辑硬伤/设定冲突）
2. **7-Layer Scan**：逐层逐段扫描并修正（禁止抽样检查）
3. **Rewrite Algorithm**：对命中的AI痕迹执行改写算法
4. **No-Poison Check**：5类毒点检测
5. **Naturalization**：对照自然化标准微调
6. **Force Check**：输出 `anti_ai_force_check: pass/fail`

### 强制门控协议
- 必须完整执行"7层规则 + 高频词库 + 终检清单"，禁止裁剪为"建议项"
- 必须按全文逐段检查执行，禁止抽样检查
- 全文章节、全部对白逐轮检查
- 章首/章中/章末都必须通过Anti-AI规则
- 若 `anti_ai_force_check` 为 `fail` → 必须回到步骤2重新扫描，直到pass
- 若命中高风险表达但因剧情必要保留，必须记录 deviation（位置+原因+代价）

### 通过阈值
- [ ] 全文不得保留未处理的三段式枚举模板句
- [ ] 全文命中的高风险表达均已改写或记录deviation
- [ ] 至少完成3处"抽象句→行为句"改写（全文范围）
- [ ] 章末存在可感知的下一步压力或未闭合问题
- [ ] 对话存在真实冲突意图，不是信息宣讲
- [ ] No-Poison五类毒点红线已检查
- [ ] 修改后未破坏设定与大纲

### 润色红线（不可突破）
- 不改剧情走向（大纲即法律）
- 不改设定物理边界（设定即物理）
- 不删除关键伏笔
- 不强行改写角色关系基线
