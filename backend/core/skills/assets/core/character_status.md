## v2.1 Skill Contract

### 适用场景
- 章节涉及角色位置、伤势、能力、关系、物品或情绪状态变化。
- 生成前后需要检查角色状态连续性。

### 输入上下文要求
- 角色档案、上一章状态、当前章节事件、关系变化和关键道具。
- 待确认知识候选中的角色状态变化。

### 输出要求
- 输出角色状态变化摘要和冲突提示。
- 状态应能写入候选知识或角色档案历史。

### 禁止事项
- 禁止忽略上一章伤势、位置、物品或关系变化。
- 禁止未经用户确认直接写结构化知识库。

### 联动触发条件
- 角色状态/人物状态/状态变化/受伤/能力/关系变化/当前位置/物品
- 角色冲突/状态冲突

## 角色状态追踪

章节之间最容易出问题的，不是大设定，而是角色状态悄悄断裂。

### 追踪维度
- 位置：角色此刻在哪里，如何移动到这里。
- 情绪：当前情绪、来源、强度、变化趋势。
- 能力：体力、法力、技能、受伤与恢复。
- 关系：与其他角色的信任、敌意、合作或裂痕。
- 物品：携带物、关键道具、失去或新增资源。

### 写作检查
- 角色是否能在逻辑上到达当前地点？
- 上一章的伤势、失去的物品、关系变化是否仍有效？
- 当前状态是否已写入记忆，供后续章节复用？
