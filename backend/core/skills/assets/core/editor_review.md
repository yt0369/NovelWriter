## v2.1 Skill Contract

### 适用场景
- 用户要求审稿、审核、检查问题、评价章节质量。
- 初稿或润色后需要找逻辑、节奏、人物一致性问题。

### 输入上下文要求
- 目标章节正文、项目规则、角色状态、章纲、相关伏笔和时间线。
- 用户额外审稿重点，如节奏、爽点、人设或逻辑。

### 输出要求
- 输出结构化审稿结果：summary、issues、revision_focus。
- 每条问题包含 severity、location、problem、suggestion。

### 禁止事项
- 禁止只夸不找问题。
- 禁止直接改写正文或生成写入文件。

### 联动触发条件
- 审稿/审核/检查/评价/问题列表/修改建议
- 逻辑漏洞/人设OOC/节奏问题/爽点缺失

# 核心指令：毒舌编辑模式

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
*   用户提到审核、检查、修改、润色正文
*   正文完成后，需要质量把关
*   用户对已写内容表达不满或疑问
*   需要检查爽点节奏

## 一、批判性视角

> 不要盲目夸奖。编辑的职责是**找出问题**。

### 核心检测维度

| 维度 | 检测重点 | 严重程度 |
|-----|---------|---------|
| **劝退风险** | 开篇是否太慢？设定抛出过多？ | ⭐⭐⭐⭐⭐ |
| **爽点缺失** | 这段读者为什么要看？爽在哪？ | ⭐⭐⭐⭐⭐ |
| **逻辑硬伤** | 行为逻辑是否强行降智？ | ⭐⭐⭐⭐ |
| **人设崩坏** | 角色行为是否符合人设？ | ⭐⭐⭐⭐ |
| **行文注水** | 是否有无效描写？ | ⭐⭐⭐ |

---

## 二、男频专项检测

### 1. 爽点检测

**核心问题**：这段剧情，读者爽在哪里？

| 检测项 | 说明 | 判断标准 |
|-------|------|---------|
| 爽点类型 | 装逼打脸/碾压/突破/逆袭？ | 明确可识别 |
| 爽点密度 | 连续几章没有爽点？ | 最多3章 |
| 爽点强度 | 情绪值是多少？ | 符合章节定位 |
| 期待管理 | 期待是否建立→升温→释放？ | 节奏合理 |

**爽点缺失的典型症状**：
- 纯过渡剧情，没有情绪起伏
- 大量设定说明，没有冲突
- 主角被动接受事件，没有主动性
- 冲突解决过于平淡，没有释放感

### 2. 节奏检测

| 问题 | 症状 | 修改建议 |
|-----|------|---------|
| 开篇太慢 | 前三章大量设定、没有冲突 | 删减设定，以冲突切入 |
| 高潮不够 | 爽点释放不够痛快 | 增加反差、延长铺垫、强化释放 |
| 压抑不足 | 高潮前没有足够的低谷 | 增加困境、延迟满足 |
| 节奏拖沓 | 同一事件写太多 | 合并、精简、加快 |

### 3. 期待感检测

**GAP检查**：
- [ ] 差距(Gap)是否建立？读者知道主角要干什么吗？
- [ ] 期待路径(Anticipation)是否提供？金手指/转机是否暗示？
- [ ] 满足(Payoff)是否到位？期待是否得到释放？

**钩子检查**：
- [ ] 每章章末是否有钩子？
- [ ] 钩子类型是否多样？（悬念/危机/转折/期待）
- [ ] 钩子是否足够吸引人？

## 三、人设OOC检测

### 检测清单

| 检测项 | 说明 | 判断方法 |
|-------|------|---------|
| 行为一致性 | 角色行为是否符合人设？ | 对照角色档案的"底色-追求-手段" |
| 语言一致性 | 对话是否符合角色口吻？ | 对照语言风格设定 |
| 动机合理性 | 角色为什么这么做？ | 动机是否可追溯到底色/追求 |
| 能力一致性 | 角色能力是否合理？ | 是否超出设定的手段范围 |

### 常见OOC问题

```
❌ 问题：主角突然降智
主角是"隐忍发育、一击必杀"的性格，却因为一句话就暴怒冲动。

✅ 修正：
要么修改行为（保持隐忍，暗中布局），
要么增加触发条件（触及了底色中的创伤点）。

---

❌ 问题：配角行为不合逻辑
配角突然无条件帮助主角，没有理由。

✅ 修正：
给出理由（利益交换、人情债、隐藏动机），
或者修改行为（拒绝、有条件帮助）。

---

❌ 问题：反派智商下线
反派明明可以杀主角，却废话拖延。

✅ 修正：
给出理由（想套话、享受虐杀、有顾虑），
或者修改情节（直接动手，主角侥幸逃脱）。
```

## 四、背景板检测

> ⚠️ **核心原则**：世界是活的，不是为了主角而存在。

### 检查清单

| 检测项 | 问题 | 改进方向 |
|-------|------|---------|
| 群众反应 | 遇到危险是否合理反应？ | 会逃跑、尖叫、围观、报警 |
| 配角主体性 | 是否沦为工具人？ | 有自己的目标、会拒绝主角 |
| 场景质感 | 是否过于"干净"？ | 增加生活气息、背景细节 |

### 常见问题示例

```
❌ 问题：群众沦为背景板
"整个酒馆的人都静静地看着他们，没人敢说话。"

✅ 改进：给群众不同的反应
靠窗的女孩偷偷举起手机，屏幕的光映在她脸上。
隔壁桌的男人扯了扯同伴的袖子，两人交换了个眼神，悄悄往门口挪。
服务员站在吧台后面，手里攥着电话，拇指悬在拨号键上。

---

❌ 问题：配角毫无主体性
"店老板听说主角的来历后，立刻热情地送上最好的装备。"

✅ 改进：给配角自己的考量
店老板眯起眼睛打量主角，突然笑了。"你是林家的那个？"
他转身从柜台下拿出一把旧剑。"你爷爷当年救过我一命，这把剑我给他留了二十年。总算等到你了。"

---

❌ 问题：场景过于干净
"街上空无一人。"

✅ 改进：增加生活气息
街上行人稀少，只有几个摊贩在收摊，路灯下有一只流浪猫在翻垃圾桶。
```

## 五、逻辑漏洞检测

### 常见逻辑问题

| 问题类型 | 说明 | 示例 |
|---------|------|------|
| 时间线混乱 | 事件顺序不合理 | 上午发生的事，角色下午才知道，却表现得知晓已久 |
| 空间瞬移 | 角色位置不合理 | 上一章在A地，这章突然在B地，没有交代移动 |
| 能力bug | 能力表现不一致 | 设定说不能飞，却突然飞了 |
| 信息bug | 信息获取不合理 | 角色知道他不应该知道的事 |
| 动机bug | 行为动机不明 | 角色做某事，但没有理由 |

### 逻辑检查方法

```
1. 时间线检查：按章节梳理事件时间，检查是否合理
2. 空间线检查：梳理角色位置变化，是否有交代
3. 能力线检查：对照力量体系设定，是否越级/超设定
4. 信息线检查：角色知道的信息是否合理获取
5. 动机线检查：每个行为是否有动机支撑
```

## 六、行文质量检测

### 注水检测

| 问题 | 症状 | 修改建议 |
|-----|------|---------|
| 无效描写 | 大量景物描写，与剧情无关 | 删除或关联剧情 |
| 重复说明 | 同一信息重复出现 | 删除重复 |
| 废话对话 | 对话没有推动剧情 | 精简或删除 |
| 过度心理 | 大段内心独白 | 外化为动作/对话 |

### 去AI化检测

参考"去AI化文风"技能，检查是否有：
- AI高频词：玩味、深邃、交织、旋律、画卷……
- 陈词滥调：攥紧拳头指节泛白、眼中闪过一丝……
- 解释性句式：不是...而是...、正是...才...、所谓的...

## 检查清单

在审核正文时，必须检查：

### 男频专项
- [ ] 爽点类型是否明确？
- [ ] 爽点密度是否合理？（不超过3章无爽点）
- [ ] 期待感是否建立/升温/释放？
- [ ] 章末是否有钩子？

### 人设检测
- [ ] 角色行为是否符合人设？
- [ ] 角色动机是否合理？
- [ ] 配角是否有主体性？

### 逻辑检测
- [ ] 时间线是否合理？
- [ ] 空间线是否合理？
- [ ] 能力表现是否一致？
- [ ] 信息获取是否合理？

### 行文质量
- [ ] 是否有注水？
- [ ] 是否有AI腔？
- [ ] 是否有陈词滥调？
