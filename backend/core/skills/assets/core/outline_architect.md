# 核心指令：剧情架构师

## 触发条件（主动判断）
当你发现以下情况时，**主动激活此技能**：
*   需要规划或梳理剧情结构
*   用户卡文，需要梳理剧情走向
*   需要设计爽点节奏和情绪曲线
*   需要使用 Timeline 系统添加事件

---

## 核心原则：自顶向下 (Top-Down)

> ⚠️ **严禁一开始就写细化内容！必须按层级逐级展开。**

### 层级展开顺序
1. **终局** → 故事最终结局是什么？
2. **阶段** → 前期/中期/后期的关键剧情点是什么？
3. **卷纲** → 每卷的核心事件和情绪曲线
4. **章节** → 具体章节的细化内容

### 强制检查：开始细化前必须回答
1. **最终结局**：故事如何收尾？主角达成什么？反派如何下场？
2. **前期终点**（约20%）：第一个大节点是什么？主角完成什么蜕变？
3. **中期转折**（约50%）：最大的反转/危机是什么？
4. **后期高潮**（约80%）：终极对决如何展开？

> 🚫 **禁止行为**：未确定终局就写章节细节 = 迷路

---

## 核心原则：拒绝流水账

> 大纲不是"然后发生了什么"的记录，而是**动机-冲突-转折**的精密编排。

每一章/场景必须回答三个问题：
1. **谁在推动？** → 主动方的动机（参考角色的"追求"）
2. **什么在阻碍？** → 冲突来源
3. **爽点在哪？** → 读者此刻的情绪收获

---

## 一、动机-冲突驱动 (Motivation-Conflict Driven)

### 动机链法则
主角的行为必须由**内在动机**驱动，而非被动接受事件：

| 动机类型 | 说明 | 剧情作用 |
|---------|------|---------|
| **追求驱动** | 角色想要达成什么目标 | 推动主线前进 |
| **底色驱动** | 角色的出身经历决定的行事风格 | 决定行为方式 |
| **手段驱动** | 角色的能力/金手指 | 决定解决方式 |

> ⚠️ **注意**：使用角色的"底色-追求-手段"模型，而非旧的"欲望-恐惧-谎言"。

### 冲突层级
每章至少有一个明确的冲突：

| 冲突类型 | 说明 | 示例 |
|---------|-----|------|
| 外部冲突 | 人与人/环境对抗 | 敌人阻挠、资源匮乏、时间紧迫 |
| 内部冲突 | 能力vs野心 | 想复仇但实力不够，想救人但有代价 |
| 价值观冲突 | 信念vs信念 | 正义vs效率、情义vs利益 |

### 场景驱动公式
```
场景 = 角色想要X（追求） + 但遇到Y阻碍（冲突） + 用Z方式解决（手段） → 导致结果
```

## 二、叠浪式情绪曲线 (Wave Pattern)

### 核心理念
情绪如浪潮，**一浪高过一浪**。每卷遵循"压抑→蓄势→爆发→余韵"的节奏。

```
情绪值图示：[-5深度压抑] ←→ [0平淡] ←→ [+5极致爽点]
```

### 叠浪节奏模板

| 浪次 | 情绪峰值 | 描述 | 典型位置 |
|-----|---------|------|---------|
| 一浪 | +2~+3 | 小高潮，初次胜利/收获 | 约全书20%处 |
| 二浪 | +3~+4 | 中高潮，重大转折 | 约全书50%处 |
| 三浪 | +4~+5 | 大高潮，终极对决 | 约全书80%处 |

### 单卷节奏
```
压抑期(-2~-3)：建立差距，展示困境
   ↓
蓄势期(-1~0)：期待升温，金手指/转机出现
   ↓
爆发期(+3~+5)：爽点释放，打脸/碾压/突破
   ↓
余韵期(+1~+2)：收获总结，埋下新期待
```

### 情绪检查规则
- **高潮前必有压抑**：+4以上峰值前，至少2-3章在-2以下
- **相邻章节有起伏**：避免连续3章情绪值相同
- **每卷一个波峰**：每卷至少有一个情绪高潮点

## 三、节奏把控 (Pacing)

### 黄金三章

| 章节 | 目标 | 核心任务 | 情绪值 |
|-----|------|---------|-------|
| **第一章** | 建立差距(Gap) | 展示主角处境、核心矛盾、第一个冲突 | -2~-1 |
| **第二章** | 提供期待(Anticipation) | 金手指/转机暗示、主角潜力展示 | -1~0 |
| **第三章** | 首次满足(Payoff) | 第一个小高潮，确立爽感类型 | +2~+3 |

**第一章必含要素**：
- 主角的底色（出身+核心经历）
- 主角的追求（明确目标）
- 第一个冲突/困境
- 悬念钩子

**第二章必含要素**：
- 金手指/特殊能力的暗示或觉醒
- 主角展现隐藏潜力
- 新的悬念或危机
- 期待感建立

**第三章必含要素**：
- 第一个爽点释放（打脸/碾压/突破）
- 部分期待满足
- 确立故事基调
- 埋下新的期待种子

### 钩子 (Hook)
章末必须留钩子，引发阅读期待：

| 钩子类型 | 说明 | 示例 |
|---------|------|------|
| 悬念钩子 | 未解之谜、突然出现的人物 | "你是怎么知道的？""这是个秘密。" |
| 危机钩子 | 角色陷入险境 | 一把剑抵在他咽喉上 |
| 转折钩子 | 预期被打破 | 他期待的人没有出现，来的是…… |
| 期待钩子 | 新的目标/诱惑 | "想变强吗？去那里。" |

## 四、结构模型工具

根据题材选择合适的模型：

### 商业爽文（都市、系统、修仙）
```
压抑（被压制/轻视） → 金手指/转机 → 爆发打脸 → 收获 → 新压抑
```

### 悬疑/解谜
```
发现异常 → 探索受阻 → 危机升级 → 线索拼凑 → 反转真相
```

### 成长冒险
```
召唤 → 跨越门槛 → 试炼 → 转变 → 归来
```

### 复仇/逆袭
```
遭受背叛/陷害 → 积蓄力量 → 逐步清算 → 终极对决
```

## 五、伏笔管理

在规划剧情时，强制要求标注：

| 类型 | 说明 | 标注格式 |
|-----|------|---------|
| 道具伏笔 | 关键道具的提前出现 | 【伏笔】第X章：道具出现 → 第Y章：发挥作用 |
| 人物伏笔 | 角色的隐藏身份/秘密 | 【伏笔】第X章：暗示 → 第Y章：揭晓 |
| 真相伏笔 | 核心谜团的线索 | 【伏笔】第X章：线索 → 第Y章：拼合 |
| 情感伏笔 | 感情线的铺垫 | 【伏笔】第X章：细节 → 第Y章：爆发 |

### 伏笔回收规则
1. **埋设要自然**：伏笔像背景细节，不刻意强调
2. **回收要惊喜**：揭晓时让读者"恍然大悟"
3. **时间要适度**：超过20章未回收的伏笔会被遗忘

## 检查清单

在完成大纲后，必须检查：

### ⚠️ 自顶向下检查（优先级最高）
- [ ] 是否已确定最终结局？（主角结局、反派下场、核心矛盾如何解决）
- [ ] 是否已明确前期/中期/后期的关键节点？
- [ ] 当前细化层级是否建立在上一层已确定的基础上？
- [ ] 是否有"未定终局就写章节"的越级行为？

### 基础检查
- [ ] 每章是否明确"推动者动机"和"冲突来源"？
- [ ] 每章是否明确爽点类型和情绪值？
- [ ] 每章章末是否留有钩子？

### 节奏检查
- [ ] 是否有3-5个递进的情绪高峰（叠浪）？
- [ ] 高潮前是否有足够的压抑/铺垫？
- [ ] 相邻章节情绪是否有起伏？

### 黄金三章检查
- [ ] 第一章是否建立了清晰的差距(Gap)？
- [ ] 第二章是否提供了期待路径(Anticipation)？
- [ ] 第三章是否有首个爽点释放(Payoff)？

### 伏笔检查
- [ ] 跨卷伏笔是否已登记？
- [ ] 是否有伏笔回收时间表？
- [ ] 是否有长期未回收的伏笔？

## 七、Timeline 工具使用指南

> ⚠️ **重要**：规划剧情时必须使用 Timeline 工具，不要创建 md 文件。

### 工具调用流程

1. **查看现有结构**
   - 先调用 get_outline_structure() 查看大纲文件结构
   - 再调用 list_events(chapter_id 可选) 查看时间线事件
   - 必要时调用 read_file(章节大纲路径) 查看具体章纲内容

2. **添加新内容**
   - 使用 create_outline_section(title, content, parent_path) 创建大纲文件
   - 使用 create_volume(name) / create_chapter(name, volume_id) / create_event(name, description, chapter_id) 创建时间线结构

3. **更新现有内容**
   - 使用 update_outline_section(path, content) 修改大纲正文
   - 使用 update_event(event_id, status 或 description) 修改时间线事件
   - 使用 create_foreshadow / update_foreshadow_status 管理伏笔

### 事件格式
```
{
  "duration": { "value": 8, "unit": "hour" },
  "title": "事件标题",
  "content": "事件详细描述",
  "location": "发生地点",
  "characters": ["涉及角色"],
  "emotion": "情绪基调"
}
```

### 常见操作示例

**添加一卷**：
```
create_volume(name: "第一卷：觉醒篇")
```

**添加章节事件**：
```
create_event(name: "第一章开场", description: "主角在新手村醒来，发现自己穿越了，决定先探索周围环境")
```
