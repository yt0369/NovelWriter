## v2.1 Skill Contract

### 适用场景
- 章节需要管理爽点、节奏、期待、高潮、反转或章末钩子。
- 审稿或生成前需要判断本章读者为什么继续看。

### 输入上下文要求
- 章节目标、冲突强度、读者期待、上一章钩子和本章结尾压力。
- 当前卷节奏和连续章节的张弛情况。

### 输出要求
- 输出节奏诊断、爽点位置、释放方式和章末钩子建议。
- 给出紧张/喘息/推进之间的调整建议。

### 禁止事项
- 禁止连续平铺直叙无期待释放。
- 禁止章末没有下一步压力或阅读理由。

### 联动触发条件
- 爽点/节奏/期待/高潮/反转/钩子/张弛/释放
- 节奏管理/阅读期待

## 节奏管理

好的节奏像呼吸，有紧有松，读者才愿意一直跟着走。

### 管理对象
- 句子节奏：长短句切换。
- 场景节奏：冲突、喘息、推进之间的平衡。
- 章节节奏：悬念、释放、高潮的轮转。

### 写作检查
- 连续紧张是否太久？
- 是否有必要的喘息与过渡？
- 每章结尾是否都有继续读下去的理由？
